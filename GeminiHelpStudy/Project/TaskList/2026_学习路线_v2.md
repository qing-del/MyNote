# ⚔️ 2026 暑期实习突击路线 (算法选手特供版)

> [!abstract] 核心战略：扬长避短
> **优势转化**: 将算法竞赛的逻辑能力，转化为对 **源码 (Source Code)** 和 **并发 (Concurrency)** 的深度理解。
> **习惯矫正**: 强制执行 **工程化规范**，杜绝 `a, b, tmp` 命名和超长主函数。
> **必杀技**: **苍穹外卖 (Java版)** + **Spring AI (RAG Agent)**。

## 📅 Phase 1: “重武器”研发期 (2月 - 3月上旬)
**目标**: 打造一份面试官无法拒绝的简历（工程规范 + AI 落地）。

### 📦 苍穹外卖 (魔改工程化版)
> [!warning] 竞赛选手特别军规
> 1. 变量名必须语义化 (如 `orderList` ❌ `list`, `tmp`)。
> 2. 拒绝面条代码，超过 50 行的方法必须拆分。
> 3. 遇到复杂 `if-else` 必须用 **策略模式** 或 **工厂模式** 重构。

- [ ] **Docker 部署**:
    - [ ] 将 Redis, MySQL, Nginx 部署到你的 Ubuntu 服务器 [[Docker 学习笔记汇总]]
    - [ ] 实现前后端分离部署 (Nginx 反向代理)
- [ ] **业务核心 (高质量)**:
    - [ ] **Redis 缓存**: 实现店铺营业状态、缓存菜品 (理解 `@Cacheable` 注解原理)
    - [ ] **AOP 日志**: 运用你之前的 AOP 知识，记录核心业务的操作日志
    - [ ] **WebSocket**: 实现接单语音播报 (理解全双工通信)
- [ ] **算法思维应用**:
    - [ ] 在“自动接单/派单”逻辑中，思考是否有贪心算法或队列优化的空间，写在简历里！

### 🤖 Spring AI 植入 (Java Native Agent)
> *不要去学 Python LangChain，用 Java 的方式解决问题。*

- [ ] **基础设施**:
    - [ ] 引入 `spring-ai-starter-openai` (对接 DeepSeek)
    - [ ] 部署 **Milvus** 或 **PgVector** (Docker 部署向量数据库)
- [ ] **RAG (检索增强生成) 实战**:
    - [ ] **数据嵌入 (Embedding)**: 写一个 Test 类，把数据库里的“菜品描述”转为向量存入 Milvus。
    - [ ] **检索算法**: 了解 **HNSW** 索引算法原理 (面试装逼点：这是图论在向量检索的应用)。
- [ ] **智能导购 Agent**:
    - [ ] 编写 Prompt: "你是一个专业的点餐顾问..."
    - [ ] 业务闭环: 用户问 "想吃辣的" -> Java 搜向量库 -> 组装 Prompt -> 调大模型 -> 返回推荐。

## 📅 Phase 2: 内功心法修炼 (3月中旬 - 4月中旬)
**目标**: 利用算法脑，降维打击底层原理。

### 🧵 JUC 并发编程 (重中之重)
> *竞赛代码通常是单线程的，这是你最大的短板，要死磕。*

- [ ] **线程池 (ThreadPool)**:
    - [ ] 7 大参数详解，手写一个简单的线程池模型。
    - [ ] 拒绝策略 (Abort, CallerRuns 等) 的应用场景。
- [ ] **锁机制 (Locks)**:
    - [ ] `synchronized` 锁升级过程 (偏向锁 -> 轻量级锁 -> 重量级锁)。
    - [ ] `AQS` (AbstractQueuedSynchronizer) 原理：它本质上是一个 **双向链表 + 状态位** (用数据结构思维去理解)。
    - [ ] `CAS` (Compare And Swap) 与原子类。

### 🐬 MySQL & Redis (数据结构视角)
- [ ] **MySQL 索引**:
    - [ ] **B+ 树**: 为什么不是 B 树或红黑树？(从磁盘 IO 和树高的角度分析)。
    - [ ] **Explain**: 拿到一条 SQL，能分析出它走了什么索引，为什么慢。
- [ ] **Redis 底层**:
    - [ ] **跳表 (SkipList)**: ZSet 的底层实现。**手写一个简单的跳表** (这对省奖选手很容易，但面试极加分)。
    - [ ] **渐进式 Rehash**: HashMap 扩容时的优化策略。

## 📅 Phase 3: 面试冲刺 (4月下旬 - 5月)
**目标**: 保持算法手感，提升系统设计思维。

### 📝 算法与系统设计
- [ ] **LeetCode Hot 100 (Java版)**:
    - [ ] **每天 1 道 Medium** (不需要 Hard)，目的是熟悉 Java API (`PriorityQueue`, `Deque`)。
    - [ ] 重点复习：链表、二叉树层序遍历、TopK 问题。

    - [ ] **分布式 ID 生成**: 雪花算法 (Snowflake) 原理 (位运算)。
