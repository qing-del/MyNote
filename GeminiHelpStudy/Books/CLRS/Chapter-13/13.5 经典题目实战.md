## 1. 区间排序与界限探测 (Medium)

在哈希表中（Chapter 11）我们仅能精准命中目标 $O(1)$。但如果我们要寻找“比给定目标稍微大一点点的值”，哈希表只能抓瞎。而这便是红黑树出场的绝对主场。

### [LeetCode 729. 我的日程安排表 I (My Calendar I)](https://leetcode.cn/problems/my-calendar-i/)
- **核心难题**：设计一个日历程序。你只能接获不重叠的日历邀约，如果新给进来的起止时间 `[start, end)` 和之前的产生了冲撞就拒绝。
- **暴力解痛点**：每次遍历 $O(N)$，面对 $1000$ 个日程表最慢退化为 $O(N^2)$。
- **解题模板：巧妙调用 TreeMap 界限 API**
  1. 将起止时间映射入 Java 的 `TreeMap<Integer, Integer> calendar`。起是键（Key），止是值（Value）。
  2. 当一个新预约 `[start, end)` 来时我们只要去寻找**树中离这个 `start` 最近的两个记录（一上一下，一前一后）**。
  3. `floorKey(start)`：返回最大的一个比 start 还要小的日程钥匙（前驱）。
  4. `ceilingKey(start)`：返回最小的一个比 start 等于或大的日程钥匙（后继）。
  5. 最核心：检查 `floorEntry` 的结束时间是否 $> start$；检查 `ceiling` 是否 $< end$。由于底层依靠红黑树的树结构搜寻，只需 $O(\log N)$ 即可防秒碰撞！

```java
class MyCalendar {
    TreeMap<Integer, Integer> calendar;
    public MyCalendar() {
        calendar = new TreeMap();
    }
    public boolean book(int start, int end) {
        Integer prev = calendar.floorKey(start); 
        Integer next = calendar.ceilingKey(start); 
        if ((prev == null || calendar.get(prev) <= start) &&
            (next == null || end <= next)) {
            calendar.put(start, end); // O(log N) 的插入
            return true;
        }
        return false;
    }
}
```

## 2. 滑动窗口与动态最值 (Hard)

### [LeetCode 218. 天际线问题 (The Skyline Problem)](https://leetcode.cn/problems/the-skyline-problem/)
- **题目难点**：有无数矩形大楼交错在这个二维坐标系上，求出外围的天际线转折坐标角点！
- **解题模板：扫描线法 + 红黑树去重排序维护**
  1. 把所有建筑物的顶边拆分为左点线（作为进入事件入场）和右点线（作为离开事件出局），沿着 $X$ 轴进行扫描推进。
  2. 我们需要一个容器，在这个扫描过程中，不断的装入高度，或者移除（也就是你大楼已经划走了）高度，并且**任何时刻我们必须在一瞬间知晓这里面目前最高的高度是多少**！
  3. 利用大顶堆（PriorityQueue/Heap）？不行！因为在堆中查找和删除非堆顶元素的时间高达 $O(N)$。
  4. 绝伦的方案：使用 `TreeMap<Integer, Integer>`！将“大楼高度”作为 Key，“出现次数”作为 Value。不仅插入和删除（或者扣减次数）全部压缩在可观的 $O(\log N)$，使用 `treeMap.lastKey()` 连带取高度最大值时同样是完美顺滑。这就是因为它的背后存在着 13.4 所讲解的精妙的红黑树动态平衡修复。

## 3. ACM 竞赛级进阶：利用红黑树的高级统计 (Advanced)

仅仅利用 Java 给好的 `TreeMap` 存在局限。有些题目我们需要依靠**扩展红黑树（OS-Tree，Order-Statistic Tree 顺序统计树，在 CLRS 第 14 章详细扩展阐释）**。

### [求当前集合中第 K 大数字 (K-th element Queries)]
- **如果使用 Java 标准库**：你无法通过树操作立刻在 $O(\log N)$ 内拿到红黑树中排名第 $K$ 的数字。如果要获取，只能通过迭代器傻乎乎从头扫描，时间退化为 $O(N)$。因此许多算法竞赛选手被迫使用并掌握 `C++ STL` 中拓展库 `#include <ext/pb_ds/assoc_container.hpp>` 提供的手搓红黑树 `tree`（也就是所谓的 PB_DS 大黑魔法库）。
- **算法理论映射**：如果希望使用这种基于节点计数的排名机制，你需要让每个红黑树节点增加一个 `size` 字段（我包含我自己及子树一共包含多少人）。我们在进行 `LEFT-ROTATE` 等操作时，在 $O(1)$ 的时间内通过简单的公式 `x.size = x.left.size + x.right.size + 1` 顺手修正就可以达成。这就为你在高并发现场实时呈现 Top K 金融交易排行榜提供了数据结构的基础。
