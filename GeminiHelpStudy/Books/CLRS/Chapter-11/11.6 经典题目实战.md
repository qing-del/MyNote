## 1. 基础预热：以空间换时间 (Easy)

### [LeetCode 1. 两数之和 (Two Sum)](https://leetcode.cn/problems/two-sum/)
- **核心思想**：利用哈希表将 $O(N^2)$ 的暴力查找降解维 $O(N)$。每遍历到一个数字 `num`，寻找哈希表中是否存在 `target - num`。如果存在则返回，不存在则把 `num` 扔进哈希表。
- **架构映射**：算法思维类似于在电商数据库里缓存已经计算过的补集。只遍历一次，属于极简的拉链法应用（虽然 Java 会帮我们封装好 HashMap）。

### [LeetCode 242. 有效的字母异位词 (Valid Anagram)](https://leetcode.cn/problems/valid-anagram/)
- **核心思想**：**直接寻址表**思想的最佳体现！由于仅有 26 个小写字母，因此绝不需要使用沉重的 `HashMap`。直接开辟一个 `int[26]` 的数组即可充当极高速的 Direct-address Table。

## 2. 进阶核心：序列查找与前缀和映射 (Medium)

### [LeetCode 128. 最长连续序列 (Longest Consecutive Sequence)](https://leetcode.cn/problems/longest-consecutive-sequence/)
- **题目难点**：要求时间复杂度必须是严格的 $O(N)$，不能排序之后找。
- **解题模板：去重后寻找子序列起点**
  1. 将所有数字装入 `HashSet` 起到去重和 $O(1)$ 查找的作用。
  2. 遍历集合中的每一个数字 `x`：只有当 `HashSet` 中不存在 `x - 1` 时，我们才确信 `x` 是一个连续序列的**源头（起点）**。
  3. 顺藤摸瓜寻找 `x+1`, `x+2` ... 挑战最长记录。
- **考察点**：哈希表的平均查找与命中效率。

### [LeetCode 560. 和为 K 的子数组 (Subarray Sum Equals K)](https://leetcode.cn/problems/subarray-sum-equals-k/)
- **题目难点**：数组中有负数，滑动窗口失效。如何高效求解？
- **解题模板：前缀和 + HashMap**
  - 利用等式：`prefixSum[j] - prefixSum[i-1] = K` 推导转化得到所需的目标：我们要寻找之前出现过的 `prefixSum[i-1] = prefixSum[j] - K`。
  - 在遍历并累加前缀和的过程中，利用一个 `HashMap` 记录每一个**前缀和出现过的次数**。
  - 把时间复杂度从纯前缀和计算的 $O(N^2)$ 砍到 $O(N)$。

## 3. 复杂应用：哈希与其它结构的共舞 (Hard)

### [LeetCode 146. LRU 缓存机制 (LRU Cache)](https://leetcode.cn/problems/lru-cache/)
- **题目难点**：既要 $O(1)$ 获取元素，又要 $O(1)$ 的驱逐历史数据（有序性）。
- **解题模板：复合数据结构（哈希表 + 双向链表）**
  - **哈希表** (`HashMap<Integer, Node>`)：用来应对 $O(1)$ 的精确查询（类似 11.2 章提及槽位的直接定位）。
  - **双向链表** (`Double-linked List`)：用来管理缓存被访问的次序时间轴。
  - 这正是 Java `LinkedHashMap` 的底层数据结构骨架雏形。

### [LeetCode 380. O(1) 时间插入、删除和获取随机元素](https://leetcode.cn/problems/insert-delete-getrandom-o1/)
- **核心思想**：变种数据结构。需要一个哈希表记录值对应的位置索引，配合一个动态数组（用于做到 `Math.random` 平均选择）。遇到删除时，为了不破坏数组的紧凑性，把数组最末尾的元素移动到被删元素的位置（Swap 操作），并配合哈希表更新索引。

## 4. ACM 竞赛级进阶：字符串哈希算法模板 (Advanced)

仅仅利用 Java 给好的 `HashMap` 并不足以触及算法竞赛的底色。真正的硬核在于——**手写开放寻址探查，或者手写字符串的多项式哈希处理结构。**
以下是 Luogu 的经典字符串处理模板。

### [洛谷 P3370 【模板】字符串哈希](https://www.luogu.com.cn/problem/P3370)
- **核心思想**：Rabin-Karp 滚动哈希的思想。
- **模板精要**：如何通过我们 `11.3 散列函数设计` 中的“除法/乘法散列”，把一个几百个字符组成的巨长串投射为一个 `long` 或者 `BigInteger`？
```java
// 核心模板：多项式散列 (Polynomial Rolling Hash)
long hash = 0;
int base = 131;        // 选择质数基底，类似于我们在 11.3 讲的常数选择
long mod = 1_000_000_000_000_000L + 7; // 防止 long 型爆炸超域

for(int i = 0; i < str.length(); i++) {
    hash = (hash * base + str.charAt(i)) % mod;
}
return hash;
```
> **架构师关联**：当海量大文本判重不仅是在单机，而是涉及到分布式节点之间的冗余检查时（比如各种杀毒云平台的病毒指纹判定），Rolling Hash 这种自底向上的高通量特征检测手段依然在焕发耀眼的生命力。
