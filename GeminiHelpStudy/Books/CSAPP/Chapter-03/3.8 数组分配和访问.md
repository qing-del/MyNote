## 数组的基本原则与连续化存储

C 语言中的数组是一种将**标量数据（相同类型）聚集成更大数据类型**的直接方式。
对于数据类型 `T` 和整型常数 `N`，声明如下：
```c
T A[N];
```
它的机器级执行意义只有极其纯粹的两条铁规：
1. 它在内存中分配了一个由 `L * N` 个字节的**极其平整且连在一起不可割裂断开（Contiguous）**的空白内存区。（其中 L 是数据类型 T 在当前汇编层级的标准占用长度，比如 int 是 4 字节长，所以一个数组包含 5 个格子就是 20 字节。这就是前面讲过为什么一定要明确知晓各种修饰符大小字长的关键！）。
2. 它引入了核心标识符变量 `A`！。`A` 它既不是数组本身也不是什么高端封装的大集合：它是且仅仅是这个**片区开始的一头处第一块内存的绝对指针。**

## 指针运算与数组：其实是同一个东西

在计算机底层中，对于一个给定的数组和指针：**加一意味着跳过一整个 L 个字节去触及下一个槽位！！**
即：`A[i]` 取数据的能力完全等价于你写出的极其暴力的 `*(A + i)`。

> [!important] 还记得神级寻址公式吗：`Imm(Rb, Ri, s)`！数组完全是靠其撑场！
> 假定数组基址存在 `%rdx` 中，也就是作为基址寄存器 `Rb`。你要访问索引的标号 `i` 设其存在 `%rcx` 内，也就是作为跑表参数 `Ri`。你要访问的是四个字节作为基础模块的一个巨无霸系统（L = 4），也就是说设置成极度切合的比例缩放子参数为 `s=4`。
> 则只需要极其直白的一句 `movl (%rdx, %rcx, 4), %eax`！就完成了一次对目标 `A[i]` 进行完美的数组解引用寻址并送入暂存处 `%eax`！如果还想查 `A[i+2]` 这种带外层包裹位移，只需要去设外摆的常数常量偏移置数位 `Imm` 为 8 即可完美解。

## 多维数组！内存只有线性的悲哀

当你开辟了一个 `T D[R][C]` 的矩形大结构！
内存是一条线拉通的一道细长带子，没有真正的二次元坐标平面给它摊开。所以在 C 里的做法是用所谓的 **“行优先（Row-Major）原则”** 把它生硬地拍平去。
先塞这块多维数据表的第一行的 `0` 到 `C-1` 号元素，再顺着开始塞入第二行，第三层……一直填平为止。

想要精准在这段压扁化的一条带上找出原有的 `D[i][j]` 这个立体坐标的实际平面降维偏移，必须要算出这条长串的最优计算规律法宝为：
**最终平滑定位点偏移距离（Address） = `基点地址 + 占宽长 L * (i * C行长度横向跨步量 + j向右挪动几个格子)`**
即必须要拿你的所处高度先乖乖乘上每长条能装下个的横向空间数值，然后再加上你这一段右挪偏移格数。

## 多级数组结构（嵌套指针解法）

为了避免要乘上像多维数组那极其繁杂甚至不规律大小带来的算术难度耗时（比如 Java 就是）。
产生了另一种多层集合解法结构：也就是生成了一份包含一堆指向目标数据各个碎片分区的数组总表！！
里面每一个 `int *A[]` 的存储表栏杆全都保存挂住了一个对应内存中存放各个杂乱碎短排长小队列的首地址的门牌索引！

此时如果你访问这团散队中的 `A[i][j]` 。它的底层必须要被迫执行极其割裂的**两次解引用读取调用**！
1. 沿着表单通过 `基址A + i * 8字节` 发现且深入找到那一条特定目标记录着某子短街门牌起点的目标表框信息，**取出解去那层该街的起始门限**（这是第一次内存查抄）。
2. 将取出来的起跑开始门限地址再加上最终落脚点：`刚才拿出的起始地址 + j * 4字节(也就是每格占据宽数)` 去执行**第二层查阅获取提取最终元素**（这是第二趟内存抓读）。

## 💡 架构师视角映射

> [!info] 与 Java 后端的联系

**Java 对于所有数组绝对安全的根基与代价**：
- 在上面的全段汇编中，我们可以清晰看出 C 或者 C++：它全是通过纯原算数位移暴力平空拿数据！！！这里没有一个人查岗来拦住你说 `i` 可不能大于前面说的尺寸限制 `N`！这也就是一切段错误越界以及攻击注入崩溃的灾难发生地。
- Java `Array` ：彻底摒弃了纯碎暴力映射规则的设计！在任何 Java 的数组进行类似于 `A[i]` 或者集合拉取的途中，在转换去给底端 CPU 执行的最终汇编上。被其自带的虚拟机会无条件强行强制注入几套比对边界的 `cmpq` 跳转异常拦截机制如果过界就立刻生成出极为出名的 `ArrayIndexOutOfBoundsException 越界超限警报反抛给外端接管。虽然对每个拉取带来指令折损耗损期但使得软件获得了长足不坏的防攻防身罩服壁保护加持！

**如果去用多维数组，Java 的真实分配方式！**
- Java 不存在上书说的绝对行优先铺平展开这种原始方案！！！
- 它的全部二维全为前文所说通过引指针列表发外包装串连出来的：`多段散链对象阵型（一级找多包分离引用地址模式）`！其产生的每一次对里面深度进行触探的二维拉出都需要一次在堆上面经历不连续极其耗时产生内存屏障损耗的高价多重解址拿调过程性能开销，绝对拖慢缓存和缓存预取等一切加速手段设施策略效力！！

## 🔭 深挖指南

> [!tip] 核心知识点与延伸阅读
>
> **本节最重要的两点**：
> 1. 数据平铺降维后要跨行寻址必须要 `L * (i * C + j)` 。
> 2. 指针嵌套的多级数组需要两次解引用（对应汇编就是两次带括号套用的 `mov` 搬迁）。
