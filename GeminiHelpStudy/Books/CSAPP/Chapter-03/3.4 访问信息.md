## 整数寄存器（16位到64位的套娃）

x86-64 芯片的脑海里，存着 16 个极其珍贵的 1 级缓存之前的VIP席位——**通用目的寄存器（General Purpose Registers）**。每个长 64 位（8 字节）。

它们的名字以 `%r` 开头。但因为 x86 的向后兼容特性，它们就像**俄罗斯套娃**一样，你可以通过不同的名字去操作这个 64 位寄存器的一小部分：
- 操作 64 位（8字节）：`%rax`
- 只操作低 32 位（4字节）：`%eax`  (早年 IA32 时期的寄存器就是这个)
- 只操作低 16 位（2字节）：`%ax`   (最早的 8086 的 16 位主寄存器)
- 只操作最低 8 位（1字节）：`%al`

```
  63                            31              15      7       0
  +-------------------------------+---------------+-------+-------+
  |                               |               |       |  %al  |  -> 1 byte
  |                               |               +-------+-------+
  |                               |            %ax                |  -> 2 bytes
  |                               +---------------+---------------+
  |                          %eax                                 |  -> 4 bytes
  +-------------------------------+---------------+---------------+ 
  |                           %rax                                |  -> 8 bytes
  +---------------------------------------------------------------+
```

其中有两个是极其神圣无法乱用的：
- `%rsp`：栈指针（Stack Pointer），永远指向当前运行时栈的栈顶。
- *还有一个不在整数寄存器之列的 `%rip` 极其重要：它是程序计数器。*

## 操作数指示符 (Operand Specifiers)

大多数汇编指令需要指出：**"我的数据从哪里来？处理完放到哪里去？"**
这个来源或目的称为**操作数（Operand）**。主要有三种：

1. **立即数（Immediate）**：在 ATT 格式的代码中，常数以 `$` 开头。
   如：`$-577` 或者 `0x1F`
2. **寄存器（Register）**：表示某个寄存器的内容。以 `%` 开头。
   如：`%rax`
3. **内存（Memory）**：将寄存器里的值当作**内存地址（指针）**，去内存的对应位置取数据。**外面套了一层括号**。
   如：`(%rax)`，代表解引用这个指针。

> [!important] 最重要的寻址模式（通用公式）
> 这个公式极其重磅，请在头脑中死死记住，你在后续碰到复杂的结构体和数组全都靠它：
> **`Imm(Rb, Ri, s)`** 
> 最终计算实际内存物理或虚拟地址的结果是： `Imm + R[Rb] + R[Ri] * s`

其中：
- `Imm` 是外挂在前面的常数偏移量
- `Rb` 是基址寄存器，指代一段区域开始的位置
- `Ri` 是变址寄存器，扮演了游标和计数器的功能
- `s` 被称为比例因子，**只能是 1、2、4 或 8**！(因为这是基本数据类型的长度)

> 类比：你现在住在 `天元小区 12栋`（基准地址 `Rb`）。每个小区的楼栋间隔长宽是 `4 步`（比例因子 `s`）。你想要走向前面 `3 栋` 的位置（游标 `Ri` 为 3）。最后走到了之后，你要精确找到那一栋的 `房门密码板上的第 5个按键`（绝对偏移 `Imm`）。所以真实坐标 = `5 + 12栋坐标 + 3 * 4步`。

## 数据传送指令 (MOV 家族)

最频繁的指令就是 **`mov` 类**，代表着数据复制！

```gas
movq  Source,  Destination   # 把 64 位（8个字节的长数据）移动走
```
- 源操作数（Source）：可以是立即数传常量、存在寄存器里的值、或是内存解引用出来的数据。
- 目的操作数（Destination）：可以是存下的寄存器、或内存。

> [!failure] x86 核心底线限制
> **两个操作数绝对不能同指内存位置！** 
> 也就是说：`movq (%rax), (%rbx)` 这一句话是报错非法的。如果要把内存中 A 区的数据搬到内存 B 区，必须经过寄存器这个中转站！中转： `A -> 寄存器 -> B`

## 压入和弹出栈数据 (push & pop)

在内存中，程序的栈是**倒着生长**的，即从高位地址向低位地址发展！

* **入栈 (`pushq S`)**：把数据 S 压入这口倒吊的井下。
  1. 先将栈顶指针寄存器 `%rsp` 向下扣划空间。即减去 8。
  2. 然后把 S 写到那个被空出来的新地址上。
* **出栈 (`popq D`)**：把当前的井底数据拿出来装给目标 D。
  1. 读取当前的 `%rsp` 地址装着的值并塞进 D 里。
  2. 把栈指针向上提，即把 `%rsp` 的数值加 8，也就是代表旧数据作废。

> 类比：想象你把一摞盘子放在桌上。栈操作就像弹夹上子弹一样，你按下一颗子弹进去（压栈），弹簧托底向下降（地址 %rsp 数值变小，向下方低地址延伸。这是所有新人都会搞反的底层思维锚点！）。取出子弹（出栈 popq），弹簧向上恢复把子弹推出来，地址变大！

## 💡 架构师视角映射

> [!info] 与 Java 后端的联系

**局部变量与栈内存**：
- 当我们在 Java 中说：对象的引用、基础数据类型局部变量全都在**线程栈**上分配时。在底层汇编这对应的就是 `%rsp` 不断上下位移（进行 `push` / `sub` 的微缩扣除空间），这套系统极为高效高速。
- 著名的 Java **StackOverflowError（栈空间溢出异常）**，在硬件级别看，就是 `%rsp` 寄存器已经往下（向低地址）伸地太深！已经挖穿了操作系统分配给这个线程在虚拟内存规划里的地址边界下限，在写如数据时触发了 OS 等级的页保护（Page Fault 段错误）！然后 JVM 把这个硬件错误捕获后抛成对应的 Exception 类型输出到你的日志中。

## 🔭 深挖指南

> [!tip] 核心知识点与延伸阅读
>
> **本节最重要的三点**：
> 1. **俄罗斯套娃寄存器格式** `%rax` 是整个全长，`%eax` 占据低频部分。
> 2. **`Imm(Rb, Ri, s)` 的核心套用公式！** 这就是数组寻址的基础。
> 3. **栈向下生长，`push` 意味着寄存器数值下降（作减法），`pop` 意味着拿走并上升（作加法）。**
>
> **深挖路径**：
> - 深刻使用 gdb 进行断点并通过 `info registers` 查看 `sub %rsp, 0x10` 以及此时对应栈地址是如何变化的。
