## stat 和 fstat 函数

应用程序可以通过 `stat` 和 `fstat` 函数来获取文件的**元数据（Metadata）**，即关于文件的信息（大小、类型、权限等），而不是文件的内容。

```c
#include <sys/stat.h>

int stat(const char *filename, struct stat *buf);
int fstat(int fd, struct stat *buf);
// 成功：返回 0，将元数据填入 buf
// 失败：返回 -1
```

| 函数 | 参数 | 区别 |
|------|------|------|
| `stat` | 文件路径名 | 通过路径访问，不需要先 open |
| `fstat` | 文件描述符 fd | 通过已打开的 fd 访问 |

```
stat vs fstat:

  方式1: stat("/tmp/data.txt", &buf);
         └── 直接通过路径获取元数据

  方式2: int fd = open("/tmp/data.txt", O_RDONLY);
         fstat(fd, &buf);
         └── 通过已打开的 fd 获取元数据
```

> 类比：`stat` 就像你在图书馆的**电脑目录系统**中搜索一本书的信息（作者、出版日期、页数），而不需要真的去书架上取出那本书（不需要 open/read）。`fstat` 则是你已经把书借出来了（已 open），顺便看看书封面上的信息。
> CS 术语：`stat` 读取的是文件的 **inode** 中存储的元数据，inode 是文件系统用于描述文件属性的核心数据结构。

## stat 结构体详解

```c
struct stat {
    dev_t     st_dev;     // 设备 ID
    ino_t     st_ino;     // inode 编号
    mode_t    st_mode;    // 文件类型 + 权限位
    nlink_t   st_nlink;   // 硬链接数
    uid_t     st_uid;     // 所有者用户 ID
    gid_t     st_gid;     // 所有者组 ID
    dev_t     st_rdev;    // 特殊设备 ID（设备文件用）
    off_t     st_size;    // 文件大小（字节）
    blksize_t st_blksize; // I/O 最优块大小
    blkcnt_t  st_blocks;  // 分配的 512B 块数
    time_t    st_atime;   // 最后访问时间
    time_t    st_mtime;   // 最后修改时间
    time_t    st_ctime;   // 最后状态改变时间
};
```

```
stat 结构体中最常用的字段:

  ┌──────────────────────────────────┐
  │ st_mode  → 文件类型 + 权限      │ ← 最重要
  │ st_size  → 文件大小（字节）     │ ← 常用
  │ st_nlink → 硬链接数             │
  │ st_uid   → 所有者 UID           │
  │ st_mtime → 最后修改时间         │ ← 常用
  │ st_ino   → inode 号             │
  └──────────────────────────────────┘
```

## 文件类型判断宏

`st_mode` 字段编码了文件类型和权限信息。Linux 提供了一组宏来判断文件类型：

```c
struct stat buf;
stat(filename, &buf);

if (S_ISREG(buf.st_mode))    // 是普通文件？
    printf("Regular file\n");
if (S_ISDIR(buf.st_mode))    // 是目录？
    printf("Directory\n");
if (S_ISLNK(buf.st_mode))    // 是符号链接？（需用 lstat）
    printf("Symbolic link\n");
```

| 宏 | 判断类型 |
|----|---------|
| `S_ISREG(m)` | 普通文件 |
| `S_ISDIR(m)` | 目录 |
| `S_ISCHR(m)` | 字符设备 |
| `S_ISBLK(m)` | 块设备 |
| `S_ISFIFO(m)` | FIFO（命名管道） |
| `S_ISLNK(m)` | 符号链接 |
| `S_ISSOCK(m)` | 套接字 |

> [!warning] stat vs lstat
> `stat` 会**跟随符号链接**，返回目标文件的元数据。
> `lstat` **不跟随符号链接**，返回链接文件本身的元数据。
> 如果你需要判断文件是否为符号链接，必须用 `lstat`。

## 💡 架构师视角映射

> [!info] 与 Java 后端的联系

**Java 的 `Files.readAttributes()` 对应 stat**：
```java
// Java NIO 获取文件元数据
BasicFileAttributes attrs = Files.readAttributes(
    Path.of("/tmp/data.txt"), BasicFileAttributes.class);

System.out.println(attrs.size());            // st_size
System.out.println(attrs.lastModifiedTime()); // st_mtime
System.out.println(attrs.isDirectory());      // S_ISDIR
System.out.println(attrs.isRegularFile());    // S_ISREG
```

**Nginx 的静态文件服务**：
- 当 Nginx 收到静态文件请求时，先 `stat()` 获取文件大小和修改时间
- `st_size` → 设置 `Content-Length` 响应头
- `st_mtime` → 设置 `Last-Modified` 响应头 → 支持 HTTP 304 缓存协商
- `S_ISDIR` → 如果是目录，返回目录列表或 403

**Spring Boot 的 `ResourceHttpRequestHandler`**：
- 处理静态资源请求时也会获取文件元数据
- `lastModified()` → 生成 `ETag` 和 `Last-Modified` 头
- 实现 HTTP 缓存机制（If-Modified-Since 协商缓存）
