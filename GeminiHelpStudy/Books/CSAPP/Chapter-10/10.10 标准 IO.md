## C 标准 I/O 库

ANSI C 定义了一套**标准 I/O 库（stdio）**，它是对 Unix I/O 的**高级封装**：

```
I/O 层次关系:

  ┌──────────────────────────┐
  │     C 标准 I/O (stdio)    │  ← fopen, fread, fprintf, fgets
  │     带缓冲的高级接口       │
  ├──────────────────────────┤
  │     RIO (CSAPP 教学用)    │  ← rio_readn, rio_readlineb
  │     健壮的中级接口         │
  ├──────────────────────────┤
  │     Unix I/O             │  ← open, read, write, close
  │     底层系统调用           │
  └──────────────────────────┘
```

标准 I/O 的核心设计：将打开的文件建模为一个**流（Stream）**，对程序员来说，一个流就是一个指向 `FILE` 结构的指针。

```c
#include <stdio.h>

// 三个预定义的流（对应 fd 0, 1, 2）
extern FILE *stdin;   // 标准输入流
extern FILE *stdout;  // 标准输出流
extern FILE *stderr;  // 标准错误流
```

> [!info] FILE 类型的本质
> `FILE` 是 C 标准库内部定义的一个结构体，它封装了：
> - 底层的**文件描述符 fd**
> - 一个**用户态缓冲区**（类似 RIO 的内部缓冲区）
> - 缓冲区的状态信息（已读/可读/剩余等）
>
> `FILE` 之于 Unix I/O 的 `fd`，就像 Java 的 `BufferedReader` 之于 `InputStream`。

## FILE 流与缓冲

每个 `FILE` 流都关联了一个**缓冲区**，用于减少底层 `read`/`write` 系统调用的次数：

```
stdio 缓冲区的工作原理:

  应用程序调用 fputc('A', stdout):
  ┌────────────────────────────────┐
  │  FILE 内部缓冲区                │
  │  [A][B][C][D][ ][ ][ ][ ]...  │
  │                ↑               │
  │             bufptr             │
  └────────────────────────────────┘
       当缓冲区满 或 遇到 '\n' 或 调用 fflush 时
       └──► write(fd, buf, count)  ← 才真正系统调用
```

## 三种缓冲类型

| 缓冲类型 | 刷新时机 | 典型用于 |
|---------|---------|---------|
| **全缓冲（Fully buffered）** | 缓冲区满时 | 普通文件的读写 |
| **行缓冲（Line buffered）** | 遇到 `\n` 换行符时 | 连接终端的流（如 `stdout`） |
| **无缓冲（Unbuffered）** | 每次写入立即刷新 | `stderr`（错误信息需立即显示） |

```c
// 示例：行缓冲的 stdout
printf("Hello ");      // 还在缓冲区中，终端上看不到
printf("World\n");     // 遇到 \n → 刷新整个 "Hello World\n" 到终端

// 示例：无缓冲的 stderr
fprintf(stderr, "Error!"); // 立即显示到终端（不等 \n）
```

> [!warning] stdout 的缓冲陷阱
> - 当 `stdout` 连接**终端**时 → **行缓冲**（遇到 `\n` 才刷新）
> - 当 `stdout` 被**重定向到文件**时 → **全缓冲**（缓冲区满才刷新）
> - 这就是为什么 `printf` 不加 `\n` 时，管道场景下输出会"延迟"
> - 解决方案：手动 `fflush(stdout)` 或 `setbuf(stdout, NULL)` 关闭缓冲

> 类比：全缓冲像**货车运快递**——攒满一车才发车；行缓冲像**公交车**——到站（换行）就开门放人；无缓冲像**出租车**——来一个乘客立即出发。
> CS 术语：缓冲策略是**吞吐量 vs 延迟**的权衡。全缓冲最大化 I/O 吞吐量（减少系统调用），无缓冲最小化输出延迟（立即可见）。

## 常用 stdio 函数

| 函数 | 说明 | 对应的 Unix I/O |
|------|------|----------------|
| `fopen` | 打开文件，返回 `FILE *` | `open` |
| `fclose` | 关闭流 | `close` |
| `fread` / `fwrite` | 二进制读写 | `read` / `write` |
| `fgets` / `fputs` | 行文本读写 | - |
| `fscanf` / `fprintf` | 格式化读写 | - |
| `feof` | 检查是否到达 EOF | - |
| `fflush` | 强制刷新缓冲区 | - |
| `fileno` | 从 `FILE *` 获取底层 fd | - |

```c
// fopen 使用示例
FILE *fp = fopen("data.txt", "r");   // 只读打开
char line[256];
while (fgets(line, sizeof(line), fp)) {  // 逐行读取
    printf("%s", line);
}
fclose(fp);
```

## 💡 架构师视角映射

> [!info] 与 Java 后端的联系

**Java I/O 的分层与 C 的分层完全对应**：

| C 标准 I/O | Java I/O | 说明 |
|-----------|----------|------|
| `FILE *` + 缓冲 | `BufferedReader` | 带缓冲的高级流 |
| `fd` (Unix I/O) | `FileDescriptor` | 底层系统资源 |
| `fflush()` | `flush()` | 强制刷新缓冲区 |
| `fprintf()` | `PrintWriter.printf()` | 格式化输出 |

**Spring Boot 日志刷新**：
- Logback 的 `ImmediateFlush=true` 相当于无缓冲模式 → 每条日志立即写磁盘
- 设置为 `false` 相当于全缓冲 → 提高吞吐但宕机时可能丢失最后几条日志
- 这就是**可靠性 vs 性能**的权衡
