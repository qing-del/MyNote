## 随机访问存储器（RAM）

### SRAM（静态RAM）

> 类比：SRAM 就像你的**工作台抽屉**。你打开抽屉，工具就在那里，随时取用，且断电后（关门前）你还能继续工作（需要持续供电才能保持）。

> CS 术语：SRAM 使用**双稳态触发器（bistable flip-flop）** 存储每一位，只要供电，数据就稳定保持，无需刷新。每个位需要 **6 个晶体管**。

```
SRAM 单元（双稳态）:

        VDD
         |
    ┌────┴────┐
    │  P-MOS  │  P-MOS
    └────┬────┘
         ├──────────────┐
    ┌────┴────┐     ┌───┴───┐
    │  N-MOS  │     │ N-MOS │   ← 形成"反相器对"，两个稳定状态
    └────┬────┘     └───┬───┘
         └──────────────┘
         |
        GND
```

**特性**：
- 速度极快（访问时间 ~1ns）
- 价格昂贵（6 个晶体管/bit）
- 无需刷新，功耗较低
- 用途：**CPU 高速缓存（L1/L2/L3 Cache）**

---

### DRAM（动态RAM）

> 类比：DRAM 就像一个**会漏气的气球**。你往里充气（写 1），它能保持一段时间，但因为漏气（电容自然放电），你需要每隔一段时间重新充气（刷新）。

> CS 术语：DRAM 使用 **1 个电容 + 1 个晶体管** 存储每一位。电容会慢慢放电，因此需要**定期刷新（refresh）**（通常每 ~64ms 刷新一次全部行）。

**特性**：
- 速度较慢（访问时间 ~60ns）
- 价格便宜（1 电容 + 1 晶体管/bit，密度高）
- 需要定期刷新（消耗带宽）
- 用途：**主存（Main Memory，DDR-SDRAM）**

---

### SRAM vs DRAM 对比

| 特性 | SRAM | DRAM |
|------|------|------|
| 存储单元 | 6 个晶体管（触发器） | 1 电容 + 1 晶体管 |
| 访问速度 | ~1 ns（极快） | ~60 ns（较慢） |
| 需要刷新 | ❌ | ✅ 每 ~64ms 一次 |
| 集成密度 | 低 | 高 |
| 每 bit 成本 | 高 | 低 |
| 典型应用 | L1/L2/L3 Cache | 内存条（DDR4/5） |
| 断电后数据 | 丢失（易失性） | 丢失（易失性） |

## 非易失性存储器（ROM 家族）

> 类比：ROM 就像**刻在石头上的文字**——即使断电也不消失。不同类型的 ROM 决定了"雕刻"有多难修改。

| 类型 | 全称 | 特点 | 典型用途 |
|------|------|------|----------|
| ROM | Read-Only Memory | 出厂写死，不可修改 | 早期固件 |
| PROM | Programmable ROM | 只能编程一次 | 定制芯片 |
| EPROM | Erasable PROM | 紫外线擦除，可重写 | 开发调试 |
| EEPROM | Electrically Erasable PROM | 电气擦除，字节级修改 | BIOS 芯片 |
| Flash | - | EEPROM 演进，块级擦除，大容量 | SSD、U盘、手机存储 |

> [!tip] 存储在 ROM 中的程序
> CPU 上电后第一条指令来自 ROM（如 BIOS/UEFI 固件），负责硬件自检（POST）和引导操作系统。
> 这段程序被 CPU 映射到固定地址空间（如 `0xFFFFFFF0`），称为 **固件（Firmware）**。

## 磁盘（Disk Drive）

### 磁盘的物理结构

```
磁盘物理结构（俯视 + 侧视）:

        ┌─────────── 盘片（Platter）─────────────┐
        │  ┌─────────────────────────────────┐  │
        │  │  磁道 9  磁道 8  ...  磁道 1  │  │  ← 每个盘片两面都有磁道
        │  │         ┌──────────────┐       │  │
        │  │         │  扇区(Sector)│       │  │  ← 每个磁道切成若干扇区
        │  │         └──────────────┘       │  │     默认 512B 或 4096B/扇区
        │  └─────────────────────────────────┘  │
        └─────────────────────────────────────────┘
              ↑
         读写磁头（悬浮在盘面几纳米处）
         磁臂带动磁头径向移动（寻道）

        盘片堆叠（侧视）:
        ════════════════  ← 盘片0
              ↑↓ 磁头
        ────────────────
        ════════════════  ← 盘片1
              ↑↓ 磁头
        ────────────────
        ════════════════  ← 盘片2

        柱面（Cylinder）= 所有盘片上相同磁道号的集合
```

**关键参数**：
- **盘片（Platter）**：存储介质，涂有磁性材料
- **磁道（Track）**：盘片上的同心圆环
- **扇区（Sector）**：磁道上的弧形片段，磁盘 I/O 的最小单位
- **柱面（Cylinder）**：所有盘片上半径相同的磁道集合

---

### 磁盘访问时间分析

> 类比：点外卖的过程——外卖员（磁头）需要先骑到你家附近的街道（寻道），然后等你所在的楼（旋转），最后上楼取餐（传输）。

> CS 术语：磁盘访问总时间 = **寻道时间 + 旋转延迟 + 传输时间**

$$T_{access} = T_{seek} + T_{rotation} + T_{transfer}$$

| 时间组成 | 典型值 | 说明 |
|---------|--------|------|
| 寻道时间（Seek Time） | ~3-9 ms | 磁臂移动到目标磁道 |
| 旋转延迟（Rotational Latency） | ~4 ms（7200 RPM） | 等待目标扇区旋转到磁头下方；平均 = 半圈时间 |
| 传输时间（Transfer Time） | ~0.02 ms/扇区 | 扇区内容经过磁头 |

> [!warning] 磁盘与内存的巨大鸿沟
> - DRAM 访问：~60 ns = 0.00006 ms
> - 磁盘访问：~10 ms
> - **差距约 100,000 倍（5 个数量级）**
>
> 这个鸿沟正是存储器层次结构存在的根本动机，也是 [[6.4 高速缓存存储器]] 的意义所在。

## 固态硬盘（SSD）

> 类比：SSD 就像一个**超大的 U 盘**，里面没有机械部件，完全是闪存芯片组成的电子存储。

```
SSD 内部架构:

┌────────────────────────────────────────────────────────┐
│                     SSD 控制器                          │
│  ┌──────────────────────────────────────────────────┐  │
│  │              闪存翻译层（Flash Translation Layer）│  │  ← 逻辑地址 → 物理块映射
│  └──────────────────────────────────────────────────┘  │
└──────────────────────────┬─────────────────────────────┘
                           │
       ┌───────────────────┼───────────────────┐
       ▼                   ▼                   ▼
  ┌─────────┐         ┌─────────┐         ┌─────────┐
  │  Package│         │  Package│         │  Package│    ← 多个闪存芯片并行
  │ ┌─────┐ │         │ ┌─────┐ │         │ ┌─────┐ │
  │ │Block│ │         │ │Block│ │         │ │Block│ │    ← 每块含 32~128 个页
  │ │Page │ │         │ │Page │ │         │ │Page │ │    ← 每页 4KB 或 16KB
  │ └─────┘ │         └─┴─────┘─┘         └─┴─────┘─┘
  └─────────┘
```

**SSD 的关键限制**：
1. **读写不对等**：读（~50μs）远快于随机写（~100μs）
2. **擦除单位是块（Block）**：写之前必须整块擦除，不能局部修改 → **写放大（Write Amplification）问题**
3. **擦写寿命有限**：每个闪存块约 10,000~100,000 次 P/E 循环，磨损均衡（Wear Leveling）是 SSD 控制器的重要工作

| 指标 | 磁盘（HDD） | SSD | DRAM |
|------|-----------|-----|------|
| 顺序读 | ~150 MB/s | ~3500 MB/s | ~50,000 MB/s |
| 随机读 | ~0.1 MB/s | ~500 MB/s | 极快 |
| 访问延迟 | ~10 ms | ~50-100 μs | ~60 ns |
| 持久性 | ✅ 非易失 | ✅ 非易失 | ❌ 易失 |

## 💡 架构师视角映射

> [!info] 与 Java 后端的联系

**JVM 内存 vs 存储器技术**：
- JVM 的**堆（Heap）** 和 **栈（Stack）** 存储在 **DRAM（主存）** 中
- JVM **GC** 的性能直接受 DRAM 访问速度制约
- *逃逸分析* 让短生命周期对象分配在**栈**（可能命中 CPU Cache）而非堆，减少 GC 压力，本质是利用了存储器层次结构

**数据库 & 中间件**：
- **MySQL 的 InnoDB Buffer Pool**：将磁盘页缓存在内存，避免每次查询都触发 ~10ms 的磁盘 I/O
- **Redis**：纯内存数据库，利用 DRAM 的 ns 级访问速度，天然比磁盘数据库快 5 个数量级
- **RDB 持久化 / AOF**：对应"将 DRAM 数据写回磁盘/SSD"，是 DRAM↔磁盘 层次结构的工程应用

**SSD vs HDD 对后端的影响**：
- 使用 SSD 后，随机 I/O 性能提升显著，**B+ Tree 索引页的随机读** 快了 100-1000 倍
- 但 SSD 的**写放大问题**对 WAL（Write-Ahead Log）、日志追加写设计提出了新的权衡

## 🔭 深挖指南

> [!tip] 核心知识点与延伸阅读
>
> **本节最重要的三点**：
> 1. SRAM（快/贵/无需刷新）用于 Cache，DRAM（慢/便宜/需刷新）用于主存
> 2. 磁盘访问时间远大于内存（5 个数量级差距），这是整个层次结构设计的根本驱动力
> 3. SSD 的擦写单位（块）大于读写单位（页），导致写放大，影响数据库设计决策
