## 系统调用的返回约定

Linux 系统调用在出错时遵循统一的约定：

> 类比：你打电话给银行客服（系统调用），如果办理成功，客服告诉你业务编号（正数返回值）；如果失败，客服固定回答"−1，具体原因请查询工单编号"（errno）。

**约定规则**：
- 返回值 **≥ 0**：调用成功，返回有意义的结果（如 `read` 返回读取字节数）
- 返回值 **= −1**：调用失败，并设置全局变量 **`errno`** 为错误码

```c
// 典型模式
int fd = open("file.txt", O_RDONLY);
if (fd < 0) {
    // 调用失败，errno 已被内核设置
    fprintf(stderr, "open error: %s\n", strerror(errno));
    exit(1);
}
```

## errno 机制

**`errno`** 是线程局部（thread-local）的全局整数，存储最近一次失败系统调用的错误码。

> [!warning] errno 两个重要注意事项
> 1. `errno` 只在系统调用**失败（返回 -1）** 时被设置；成功时 errno 的值无意义
> 2. `errno` 是**线程局部存储（TLS）** 的，多线程下每个线程有自己的 errno，不会互相干扰

常见 errno 值：

| errno 符号常量 | 数值 | 含义 |
|--------------|------|------|
| `EPERM` | 1 | 操作不被允许 |
| `ENOENT` | 2 | 文件或目录不存在 |
| `EINTR` | 4 | 系统调用被信号中断 |
| `EBADF` | 9 | 无效的文件描述符 |
| `ENOMEM` | 12 | 内存不足 |
| `EACCES` | 13 | 权限拒绝 |
| `EBUSY` | 16 | 设备或资源忙 |
| `EEXIST` | 17 | 文件已存在 |
| `EAGAIN` | 11 | 资源暂时不可用（非阻塞 I/O） |

## 错误处理包装函数

CSAPP 推荐为每个系统调用封装一个**错误检查包装函数**，将错误检查内嵌到调用中，保持代码整洁：

```c
// 包装 open 系统调用
int Open(const char *pathname, int flags, mode_t mode) {
    int rc;
    if ((rc = open(pathname, flags, mode)) < 0) {
        fprintf(stderr, "Open error: %s\n", strerror(errno));
        exit(0);
    }
    return rc;
}

// 包装 fork 系统调用
pid_t Fork(void) {
    pid_t pid;
    if ((pid = fork()) < 0) {
        fprintf(stderr, "Fork error: %s\n", strerror(errno));
        exit(0);
    }
    return pid;
}
```

> [!tip] Java 中的对应思想
> Java 将系统调用错误直接转化为异常（`IOException`、`FileNotFoundException` 等）。
> JVM 内部调用系统调用后，若返回 -1，就读取 errno 并包装为对应的 Java 异常抛出。
> Java 程序员无需手动检查 errno，这是 Java 对 UNIX 错误处理的优雅封装。

## 💡 架构师视角映射

> [!info] 与 Java 后端的联系

**EINTR 与重试机制**：
- `EINTR`（errno=4）：系统调用被信号打断——这对后端开发极其重要
- 某些系统调用（如 `read`、`write`、`accept`）被信号打断时返回 -1 且 errno=EINTR，需要重试
- Java NIO 的底层已处理了这种情况（自动重试），但直接使用 JNI/Native 层时需手动处理

**EAGAIN 与非阻塞 I/O**：
- `EAGAIN`（errno=11）：非阻塞 I/O 暂无数据可读 — 告知调用者"现在没数据，稍后再试"
- Netty 的 NIO 事件循环本质上就是在处理 `EAGAIN`：`epoll_wait` 等到数据就绪再读，避免忙等

- Linux 错误码完整列表 → `/usr/include/errno.h` 或 `man 3 errno`
- Java IOException 与 errno 的对应关系 → JVM 源码 `jdk/src/solaris/native/java/io/`
- EINTR 重试最佳实践 → 《Unix 网络编程》卷 1 第 5.9 节
