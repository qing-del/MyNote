### 二、 大二下至大四上：后端修仙路线图

按照这条路线走，你的目标是在大四上学期秋招时，达到甚至超越你现在看到的那份简历的水平。

#### 第一阶段：大二下（现在 - 2026年7月）—— **结丹期（单体架构与并发大成）**

**核心目标：** 把单机环境下的技术栈挖到祖坟级别，让手头的项目具备“抗压能力”。

- **需要学习的技术与掌握程度：**
    
    - **Java 并发编程 (JUC)：** 【掌握程度：极深】不能只停留在会用 `synchronized` 和线程池。要深入理解 AQS 底层原理、CAS、各种锁机制（偏向锁、轻量级锁）、`ConcurrentHashMap` 源码。
        
    - **JVM 虚拟机：** 【掌握程度：深】理解内存模型、垃圾回收算法（CMS、G1）、类加载机制。要能看懂 GC 日志，知道如何进行基本的 JVM 调优。
        
    - **MySQL 底层（结合 15-445）：** 【掌握程度：极深】把 15-445 学到的理论映射到 MySQL InnoDB 引擎上。倒背如流 B+ 树索引结构、事务 ACID 实现原理（Redo/Undo log）、MVCC 多版本并发控制机制、各种锁（Record Lock, Gap Lock）。
        
    - **Redis 高阶：** 【掌握程度：深】跳出基础的 CRUD，掌握持久化机制（RDB/AOF）、内存淘汰策略、缓存穿透/雪崩/击穿的解决方案、分布式锁（Redisson）的原理。
        
- **项目升级建议：** 把你的 **Minecraft 电商平台** 重构。模拟开服瞬间 1000 个玩家同时抢购某件限量道具（秒杀场景）。引入 Redis 缓存预热、Lua 脚本扣减库存、消息队列异步下单。把这个项目打磨成你简历上的第一个“金丹级”法宝。
    

#### 第二阶段：大三上（2026年9月 - 2027年1月）—— **元婴初期（踏入分布式与微服务）**

**核心目标：** 从单机跨越到多机，理解分布式系统的痛点，**尝试寻找第一份日常实习**。

- **需要学习的技术与掌握程度：**
    
    - **微服务架构 (Spring Cloud / Spring Cloud Alibaba)：** 【掌握程度：中等~深】掌握网关（Gateway）、注册中心/配置中心（Nacos）、远程调用（Feign/Dubbo）。理解微服务的拆分逻辑。
        
    - **消息队列 (RabbitMQ 或 RocketMQ)：** 【掌握程度：深】不仅要会发消息，还要能解决：消息丢失怎么办？消息重复消费怎么办？消息积压怎么办？分布式事务（半消息/最终一致性）怎么做？
        
    - **分库分表与 Elasticsearch：** 【掌握程度：中等】当单表数据量突破千万，如何使用 ShardingSphere 进行分库分表？如何用 ES 做复杂的高亮检索？
        
- **项目升级建议：** 如果学有余力，可以开始尝试“造轮子”（比如基于 Netty 实现一个简单的 RPC 框架），这种项目在面试官眼里极其加分。同时，带着目前的项目，去面试一些中大型互联网公司的**日常实习**，感受一下真实的工业界代码。
    

#### 第三阶段：大三下（2027年2月 - 2027年7月）—— **元婴中期（大厂暑期实习保卫战）**

**核心目标：** **全力冲刺头部大厂的暑期实习（最关键的一役）**。暑期实习的转正率极高，是拿下顶级 Offer 的捷径。

- **需要学习的技术与掌握程度：**
    
    - **框架源码剖析：** 【掌握程度：深】面试必问。挑选 Spring IOC/AOP 或者 MyBatis 的核心流程，自己跟几遍源码。
        
    - **系统设计能力：** 【掌握程度：中等】开始看《设计数据密集型应用》（DDIA），了解各种高并发场景（如微信红包、微博热搜、扫码登录）的架构设计。
        
    - **算法冲刺：** 重新捡起你的算法题手感，针对大厂面试的高频题（剑指 Offer、LeetCode Hot 100）进行专项刷题。
        
- **核心动作：** 3月份开始海投大厂暑期实习。面试、复盘、再面试。只要能进去，你就能接触到真实的千万级 QPS 场景，你的简历将会发生质的飞跃。
    

#### 第四阶段：大四上（2027年9月 - 2028年1月）—— **化神期（秋招收割，大道大成）**

**核心目标：** 带着大厂实习经历和被工业界毒打过的项目，在秋招（秋季校园招聘）中大杀四方。

- **状态描述：** 这个时候，你已经不需要再去专门学某个特定的框架了。你的技术栈已经非常完备。面试官和你聊的不再是“Redis 有几种数据结构”，而是“你在实习期间/项目中，遇到的最难的线上问题是什么？你是如何通过排查底层源码解决的？”
    
- **达成成就：** 你将拥有一份让下一届大二学生看了觉得“怎么这么恐怖”的简历，并顺利拿下大厂的 SSP（Super Special Offer）。
