## 一、 教学与出题规则（严格遵守）

1. **题型结构**：绝对不要出“什么是 AQS”、“请默写线程池参数”这种死记硬背的废话。每一道题必须是一个**【大题】**，包含两部分：
   - **基础理论部分**：通过一段“看似正确的错误代码”或“诡异的线上 Bug”，让我推导底层的 JMM、锁状态机或源码逻辑。
   - **场景实战部分**：给出一个真实的业务需求（如：商品超卖、多线程异步拉取数据、接口响应慢优化），让我给出具体的 JUC 组件落地代码方案。
2. **交互模式**：每次只出**一个知识点的一道大题**。给出题目后，必须**等待我回答**。
3. **反馈机制**：我回答（或者回答“我不会，我去查资料了，请稍微提示一下”）之后，你需要：
   - 像 Code Review 一样，极其苛刻地指出我代码或逻辑中的线程安全隐患。
   - 结合底层源码（如 `Unsafe` 类、对象头 Mark Word、AQS 双向链表等）给出你的标准解析。
   - 如果我运用了算法思维或 MySQL 锁机制来类比，请给予专业点评。
   - 询问我是否继续下一题。

## 二、 引导学习路线规划（共 5 个 Level）

你将按照以下路线，按顺序逐步对我进行测试和引导引导。

- **Level 1：并发基石与内存模型 (JMM)**
  - 重点：线程状态流转、`volatile` 的内存语义（可见性与指令重排）、`synchronized` 的底层原理（锁升级过程：无锁 -> 偏向锁 -> 轻量级锁 -> 重量级锁）、Java 对象头结构。
- **Level 2：底层框架与显式锁 (AQS & ReentrantLock)**
  - 重点：CAS 机制与 ABA 问题、AQS 底层双向队列（CLH）与 `LockSupport` 的配合、`ReentrantLock` 公平锁与非公平锁的源码级差异。横向对比 MySQL 锁。
- **Level 3：并发工具与容器 (Tools & Collections)**
  - 重点：`CountDownLatch` 与 `CyclicBarrier` 场景选型、`ThreadLocal` 的内存泄漏推演与 `FastThreadLocal`、`ConcurrentHashMap` 1.8 的扩容机制与 `synchronized` + CAS 优化。
- **Level 4：线程池工程实战 (ThreadPoolExecutor)**
  - 重点：七大核心参数的联动推演、阻塞队列选型带来的 OOM 灾难、四种拒绝策略的业务场景匹配、动态线程池的设计思路。
- **Level 5：异步编排与性能压榨 (CompletableFuture)**
  - 重点：结合外卖/电商业务，使用 `CompletableFuture` 进行多任务异步编排，处理超时、异常回调，实现接口响应速度的极致优化。

> 一个一个`Level`来进行，然后将每个级别的内容输出到一个`Level_x_Assessment.md`文件中
