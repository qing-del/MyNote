# 配置类
- `@Configuration` 标注这个是一个配置类
- `@SpringBootApplication` 这个是**启动类**的注解，同时具备组件扫描的功能，但是扫描的是启动类所在包及其子包
- `@ComponentScan` 可以通过这个注解中的`basePackages`属性来规定扫描的位置，但是规定之后默认的扫描就不生效了 使用示例可看->[[Spring AutoConfiguration原理#实现方案-1]]
- `@Import` 用于导入Bean对象类 详见->[[Spring AutoConfiguration原理#实现方法-2]]
- `@EnableConfigurationProperties` 用于自定义starter时给自定义的自动配置类加上

---

# Bean管理
- `@Autowired` 自动Bean注入 前提是这个类是Bean对象
- `@Scope` 设置 Bean 的作用域（包括单例或者多例）
- `@Lazy` 延迟初始化（对于单例） --> 直到第一次被使用
- `@RestController` Controller层通用的注释 将其标记为接口 Bean
- `@Component` 标注交给 IOC 容器管理
- `@Bean` 一般用于管理 `jar包` 类型的Bean 详见->[[Spring AutoConfiguration原理#第三方的Bean]]
- `@Conditional` 满足一定条件才注册到 IOC 容器中  详见 -> [[Spring AutoConfiguration原理#`@Conditional`注解]]

# 接口开发
- `@CrossOrigin` 准许的请求域（一般跨域请求都需要这个）
- `@RestController` 标记为对外接口，可以在括号内传入字符串设置bean名称
- `@RequestMapping` 写在Controller类上面的请求路劲，可以称之为前缀路径
- `PostMapping` 和 `DeleteMapping` 和 `PutMapping`(`PatchMapping`) 和 `GetMapping` 分别为增删改查的接口注释，以及特定请求路径
> `PatchMapping` 是部分修改的时候才使用
- `@RequestBody` 对于前端传回来`json`格式数据时解析使用
- `@PathVariable` 获取请求路径上的参数
- `@RequstParam` 应用于批量删除传入`ids=1,2,3`这种可以解析成`List<Long>`类型

# 逻辑层
- `@Service` 标注这个类为 逻辑层的Bean IOC容器就会自动进行管理
- `@Transactional` 开启事务

---

# 数据层
- `@Mapper` 标记这个接口为 Mapper 处理数据库的 IOC将其认为Mapper Bean
- `@Insert` 和 `@Delete` 和 `@Update` 和 `@Select` 对应方法 分别为增删改查
- `@Option` 可以利用里面两个属性来设置是否使用自动生成的`key`值

# 实体类
- `@Data` 标注在实体类上面即可自动生成 getter 和 setter
- `@NoArgsConstructor` 自动生成无参构造
- `@AllArgsConstructor` 自动生成全参构造
- `@JsonFormat` 规范化 `Json` 格式 可以用于`Date`

# 注解类 & AOP类
- `@Target` 设定这个注解的作用目标
- `@Retention` 设定这个注解的有效时间（比如设定为运行时有效）
- `@execution` 根据方法签名进行匹配 详见->[[SpringAOP#4.1 execution (根据方法签名匹配)|execution示例文档]]
- `@annotation` 根据注解进行匹配 详见->[[SpringAOP#4.2 @annotation (根据注解匹配) ✨ 推荐|annotation详细文档]]
